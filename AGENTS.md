# AGENTS.md

Calendar booking app built **Design First**: the API contract is the source of truth, and frontend/backend are generated/implemented from it independently.

## Contract-first workflow (critical)

The single source of truth is `main.tsp` (TypeSpec) at the repo root. The flow is:

```
main.tsp  --tsp compile-->  tsp-output/@typespec/openapi3/openapi.yaml  -->  frontend types + backend DTOs
```

- **Never hand-edit** `tsp-output/` or generated code. Change `main.tsp`, then regenerate.
- After editing `main.tsp`, run from repo root: `npm run generate`
  (= `tsp compile main.tsp` then `openapi-typescript ... -o frontend/src/types/api.ts`).
- `tsp-output/` is gitignored, so the OpenAPI yaml must be regenerated before frontend type-gen, the backend build, or running the Prism mock.

## Layout

- `main.tsp`, root `package.json` — contract + codegen only (no app code).
- `frontend/` — Vite + React 19 + Mantine + TypeScript. API types in `src/types/api.ts` (generated), typed client in `src/api/client.ts`.
- `backend/` — Kotlin + Ktor (Netty). In-memory storage only (`InMemoryStore`); no database.

## Frontend (`frontend/`)

- `npm run dev` — Vite dev server on `:5173`, proxies `/api/*` → `http://localhost:4010`.
- `npm run mock` — Prism mock API on `:4010` from the OpenAPI yaml. Run this (or the real backend on `:4010`) in a separate terminal for `dev` to work.
- `npm test` — Vitest (`vitest run`); `npm run test:watch` to watch. jsdom env, globals enabled, setup in `src/setup.ts`. Tests are colocated as `*.test.tsx`.
- `npm run build` — `tsc -b && vite build` (typecheck is part of build).
- `npm run lint` — ESLint (flat config, typescript-eslint + react-hooks).
- Stop dev servers: `lsof -ti :4010 -ti :5173 | xargs kill -9`.

## Backend (`backend/`)

- Requires JDK 21+. Use the Gradle wrapper (`./gradlew`).
- `./gradlew run` — server on `:4010` (override with `PORT=8080 ./gradlew run`).
- `./gradlew test` — Kotlin tests under `src/test/kotlin`.
- DTOs are generated from the OpenAPI yaml via `openapi-generator` into `build/generated/` (`com.booking.generated.model`); `compileKotlin` depends on `openApiGenerate`. Do not edit generated code.
- The backend reads `../tsp-output/@typespec/openapi3/openapi.yaml`, so run `npm run compile` (or `npm run generate`) at the root first if the yaml is missing/stale.
- Server wiring: `Application.kt` → `plugins/` (serialization, status pages, routing). Routing assembles services in `plugins/Routing.kt`; routes in `routes/`, business logic in `service/`.
- Config via env vars: `PORT` (4010), `OWNER_ID`, `OWNER_DISPLAY_NAME`, `OWNER_TIMEZONE`.

## Notes

- CI (`.github/workflows/hexlet-check.yml`) runs the Hexlet project check only — it does not run the lint/test suites above. Do not edit that file (autogenerated).
- Default availability is a single all-day window per day (`DefaultAvailabilityProvider`); slots are computed for a 14-day window.
